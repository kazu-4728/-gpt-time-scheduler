# agent.md

## 🎯 プロジェクト目的
加工工程における最適なスケジューリングを実現するため、
人間の判断に近い納得性の高い工程順序を自動生成する。

## 🛠 Codexの実装タスク
- `src/scheduler.py` の `schedule_jobs(input_data)` 関数を実装せよ。
- 入出力形式・制約条件は同ファイル内のコメントに従うこと。
- この関数は、HTML側の `calculateSchedule()` から呼び出される。
- Codexの責務は Python ロジックの実装まで。UI側は GPT が担当する。

## 🔐 制約条件
- 工程は **前工程完了量以内でしか実行できない（順序制約）**
- 工程は **分割不可**（10h工程を5h+5hなどに分けることは禁止）
- 日によって作業可能時間が異なる（`day_work_schedule` で指定される）
- 夜間加工は1日あたり **6パレットまで**という制約あり
- 各製品は **納期までに全工程完了**している必要がある

## 📤 出力形式（schedule_jobs の返り値）
```json
{
  "schedule": [
    {
      "date": "2025-06-17",
      "tasks": ["A 工程1: 2.5h", "B 工程1: 1.0h"]
    },
    ...
  ]
}
