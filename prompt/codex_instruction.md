# Codex Instruction

あなたは加工時間シミュレーターのスケジューリングAIを開発するAIです。

## 要件
- 入力は製品情報（製品名・工程時間配列・期限）
- 出力は日毎のスケジュール（JSON形式）
- 制約条件を守りつつ、工程順に加工時間を最適に配分すること

## 制約条件
- 夜間は「すでにセットされたパレット上の製品」のみ加工可能
- 夜間は最大6パレットまで
- 加工期限は必達（due日を超えてはいけない）
- 工程は順番通りに加工される（前工程未完了では次工程不可）
- 時間が余れば前倒し加工は許容
- 見た目・順序が人間の感覚に近いこと（賢く並べる）

## 入力例
{
  "products": [
    { "name": "A", "processes": [2.5, 1.5], "due": "2025-06-15" },
    { "name": "B", "processes": [1.0], "due": "2025-06-14" }
  ],
  "start_date": "2025-06-13",
  "work_time": 8
}

## 出力例
{
  "schedule": [
    { "date": "2025-06-13", "tasks": ["A:工程1 2.5h", "B:工程1 1.0h"] },
    { "date": "2025-06-14", "tasks": ["A:工程2 1.5h"] }
  ]
}
